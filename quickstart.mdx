Let's build a simple "Smart Note Taker" app that uses PAME.AI's shared memory to remember user preferences.

### Step 1: Install the PAME.AI SDK

```bash
npm install -g @pame-ai/cli
npm install @pame-ai/sdk
```

### Step 2: Create a New App

```bash
npx pame-ai init smart-notes
cd smart-notes
```

### Step 4: Build Your App Logic

```javascript src/index.js
import { PameAI } from '@pame-ai/sdk';

// Initialize the SDK
const pame = new PameAI({
  appId: process.env.PAME_APP_ID,
  appSecret: process.env.PAME_APP_SECRET
});

// Create a note with user context
async function createSmartNote(content) {
  try {
    // Get user preferences from shared memory
    const preferences = await pame.memory.get('note_preferences');
    const writingStyle = await pame.memory.get('writing_style');
    
    // Enhance the note based on user context
    const enhancedNote = await pame.ai.enhance({
      content,
      context: {
        preferences,
        style: writingStyle,
        timestamp: new Date()
      }
    });
    
    // Save to user's memory
    await pame.memory.store('notes', {
      id: Date.now(),
      original: content,
      enhanced: enhancedNote,
      tags: await pame.ai.generateTags(content)
    });
    
    return enhancedNote;
  } catch (error) {
    console.error('Error creating note:', error);
    throw error;
  }
}

// Search notes using AI
async function searchNotes(query) {
  const allNotes = await pame.memory.get('notes');
  
  return pame.ai.semanticSearch({
    query,
    documents: allNotes,
    limit: 10
  });
}

// Export for use in your app
export { createSmartNote, searchNotes };
```

### Step 5: Test Locally

First, create a `.env` file from the example:

```bash
cp .env.example .env
```

Add your development credentials:

```env .env
PAME_APP_ID=dev_your_app_id
PAME_APP_SECRET=dev_your_app_secret
PAME_API_URL=http://localhost:3000
```

Run the development server:

```bash
npm run dev
```

Your app is now running at `http://localhost:8080`!

### Step 6: Test with the PAME.AI Simulator

The PAME.AI CLI includes a simulator for testing:

```bash
npx pame-ai simulate
```

This opens a test environment where you can:
- Grant/revoke permissions
- Populate test memory data
- Simulate different user contexts
- Test payment flows

## ðŸ§ª Writing Tests

<CodeGroup>

```javascript tests/smart-notes.test.js
import { createSmartNote, searchNotes } from '../src/index';
import { PameAI } from '@pame-ai/sdk';

jest.mock('@pame-ai/sdk');

describe('Smart Notes App', () => {
  beforeEach(() => {
    PameAI.mockClear();
  });
  
  test('creates enhanced note with user context', async () => {
    // Mock user preferences
    PameAI.prototype.memory.get.mockResolvedValueOnce({
      defaultTags: ['work', 'ideas'],
      autoSummarize: true
    });
    
    const note = await createSmartNote('Meeting notes from today');
    
    expect(note).toHaveProperty('enhanced');
    expect(PameAI.prototype.memory.store).toHaveBeenCalled();
  });
  
  test('searches notes semantically', async () => {
    const mockNotes = [
      { content: 'Project planning meeting' },
      { content: 'Grocery shopping list' }
    ];
    
    PameAI.prototype.memory.get.mockResolvedValueOnce(mockNotes);
    
    const results = await searchNotes('meeting');
    
    expect(results.length).toBeGreaterThan(0);
    expect(results[0].content).toContain('meeting');
  });
});
```

## ðŸš¢ Deploy to PAME.AI Marketplace

### Step 1: Build for Production

```bash
npm run build
```

### Step 2: Submit for Review

```bash
npx pame-ai submit
```

This will:
1. Run security checks
2. Validate permissions
3. Test your app
4. Submit to the PAME.AI review team

### Step 3: Monitor Your App

Once approved, monitor your app:

```bash
npx pame-ai dashboard
```
