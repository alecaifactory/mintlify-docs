## Prerequisites

Before you begin, ensure you have:

<Steps>
  <Step title="System Requirements">
    - **OS**: macOS, Linux, or Windows with WSL2
    - **RAM**: 8GB minimum, 16GB recommended
    - **Storage**: 10GB free space
  </Step>
  
  <Step title="Required Software">
    - [Node.js](https://nodejs.org/) 18+ and npm 8+
    - [Git](https://git-scm.com/) 2.34+
    - [Docker](https://docker.com/) 20.10+
    - [Google Cloud SDK](https://cloud.google.com/sdk)
    - [Terraform](https://terraform.io/) 1.5+
  </Step>
  
  <Step title="Accounts">
    - GitHub account
    - Google Cloud Platform account
    - Mintlify account (for documentation)
    - NPM account (for package publishing)
  </Step>
</Steps>

## ğŸ“‹ Manual Setup

### Step 1: Clone the Repository

```bash
git clone https://github.com/ai-factory-llc/pame-ai.git
cd pame-ai
git checkout develop
```

### Step 2: Install Dependencies

```bash
npm install
```

### Step 3: Environment Configuration

Copy the example environment file:

```bash
cp env.example .env
```

Configure your `.env` file:

```env .env
# Core Configuration
NODE_ENV=development
PORT=3000
API_URL=http://localhost:3000

# Database Configuration
DATABASE_URL=postgresql://pameai:password@localhost:5432/pame_ai_dev
REDIS_URL=redis://localhost:6379

# Authentication
JWT_SECRET=your-super-secret-jwt-key-change-this
SESSION_SECRET=your-session-secret-change-this

# Mintlify Documentation
MINTLIFY_PROJECT_ID=your-project-id
MINTLIFY_ADMIN_API_KEY=your-admin-key
MINTLIFY_ASSISTANT_API_KEY=your-assistant-key

# NPM Registry
NPM_PACKAGE_API_TOKEN=your-npm-token

# Google Cloud Platform
GCP_PROJECT_ID=your-gcp-project
GCP_REGION=us-central1
GCP_SERVICE_ACCOUNT_KEY=path/to/service-account.json

# Stripe Payments
STRIPE_SECRET_KEY=sk_test_your_key
STRIPE_PUBLIC_KEY=pk_test_your_key
STRIPE_WEBHOOK_SECRET=whsec_your_secret

# Monitoring
SENTRY_DSN=your-sentry-dsn
LOG_LEVEL=debug
```

### Step 4: Database Setup

<Tabs>
  <Tab title="Docker (Recommended)">
    ```bash
    # Start PostgreSQL and Redis
    docker-compose up -d postgres redis
    
    # Run migrations
    npm run db:migrate
    
    # Seed development data
    npm run db:seed
    ```
  </Tab>
  
  <Tab title="Local Installation">
    ```bash
    # Install PostgreSQL
    brew install postgresql@15
    brew services start postgresql@15
    
    # Install Redis
    brew install redis
    brew services start redis
    
    # Create database
    createdb pame_ai_dev
    
    # Run migrations
    npm run db:migrate
    
    # Seed data
    npm run db:seed
    ```
  </Tab>
</Tabs>

### Step 5: Start Development Services

<CodeGroup>

```bash Terminal 1 - API Server
npm run dev:api
# API running at http://localhost:3000
```

```bash Terminal 2 - UI Development
npm run dev:ui
# UI running at http://localhost:3001
```

```bash Terminal 3 - Documentation
npm run docs:dev
# Docs at http://localhost:3002
```

</CodeGroup>

## ğŸ§ª Testing

### Run All Tests

```bash
npm test
```

### Run Specific Test Suites

<CodeGroup>

```bash Unit Tests
npm run test:unit
```

```bash Integration Tests
npm run test:integration
```

```bash E2E Tests
npm run test:e2e
```

```bash Coverage Report
npm run test:coverage
```

</CodeGroup>

### Testing Best Practices

1. **Write tests first** (TDD approach)
2. **Mock external services** in unit tests
3. **Use realistic data** in integration tests
4. **Test error scenarios** not just happy paths
5. **Maintain 80%+ coverage**

## ğŸ”§ Development Tools

### Code Quality

```bash
# Lint your code
npm run lint

# Auto-fix linting issues
npm run lint:fix

# Format with Prettier
npm run format

# Run quality checks
npm run quality:check
```

### Git Hooks

We use Husky for Git hooks:

- **pre-commit**: Runs linting and formatting
- **pre-push**: Runs tests
- **commit-msg**: Validates commit message format

### Commit Message Format

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
type(scope): subject

body

footer
```

Examples:
```
feat(auth): add OAuth2 support
fix(memory): resolve data persistence issue
docs(api): update authentication examples
chore(deps): upgrade to React 18
```

## ğŸŒ Local Infrastructure

### Start Local Services


### Common Issues

<AccordionGroup>
  <Accordion title="Port already in use">
    ```bash
    # Find process using port
    lsof -i :3000
    
    # Kill process
    kill -9 [PID]
    ```
  </Accordion>
  
  <Accordion title="Database connection failed">
    ```bash
    # Check PostgreSQL status
    brew services list
    
    # Restart PostgreSQL
    brew services restart postgresql@15
    
    # Check connection
    psql -U postgres -h localhost
    ```
  </Accordion>
  
  <Accordion title="Redis connection error">
    ```bash
    # Check Redis status
    redis-cli ping
    
    # Restart Redis
    brew services restart redis
    ```
  </Accordion>
  
  <Accordion title="npm install fails">
    ```bash
    # Clear npm cache
    npm cache clean --force
    
    # Delete node_modules
    rm -rf node_modules package-lock.json
    
    # Reinstall
    npm install
    ```
  </Accordion>
</AccordionGroup>
